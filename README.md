# 3GPP Research

3GPP RAN1（MIMO・チャネルコーディング）の技術動向を体系的に調査し、
標準必須特許（SEP）のアイデアに繋がる知見を蓄積するための個人リサーチリポジトリ。

公開情報のみを扱う。

## 対象領域

### MIMO
- Cell-Free Massive MIMO / Ultra-Massive MIMO
- RIS（再構成可能インテリジェント表面）
- AI ベース CSI フィードバック圧縮・予測
- Near-Field XL-MIMO / Sub-THz MIMO

### チャネルコーディング
- Semantic Communications / DJSCC
- GRAND（ユニバーサルデコーディング）
- AI ネイティブチャネルコーディング
- Ambient IoT 向け軽量符号化

## 調査フレームワーク

すべてのトピックを以下の **6軸** で分析する:

| 軸 | 問い |
|:---|:---|
| 技術レイヤー | 物理層のどこに位置し、他レイヤーとどう結合するか |
| 世代 | 5G Advanced / 6G のどのフェーズで標準化されるか |
| 価値 | スループット以外にどんな KPI を改善するか |
| 市場 | どのセグメント・地域で最初に展開されるか |
| 普及要因 | 採用を左右するボトルネックは何か |
| IP | 知財ポジションの現状と機会はどこか |

さらに、状況に応じて**分析レンズ**（政治経済・標準化ダイナミクス）と
**ペルソナ**（通信事業者・チップベンダー・OEM・規制当局など）を適用する。

## 核心となる原則

1. **一次情報の出典を必ず記録する** — URL・文書番号・アクセス日を明記
2. **論文の理想仮定 vs 3GPP 実装制約を常に意識する** — ギャップにこそ進歩性が潜む
3. **過去世代の成功・失敗と対比する** — 5G の教訓を 6G に反映
4. **記述は Why / What / How の3層で分ける**
5. **不明確な点は `[要確認]` とマークする**
6. **Next Steps を必ず残す** — 具体的なアクションとして記録

## Claude Code によるワークフロー

本リポジトリは [Claude Code](https://claude.com/claude-code) をリサーチアシスタントとして活用する。

### スキル（調査・分析）

| コマンド | 用途 |
|:---|:---|
| `/survey-topic` | トピックを6軸で体系調査し `10_topics/` に記録 |
| `/analyze-gap` | 学術/3GPP/実装制約のギャップを3バケットで可視化 |
| `/success-pattern` | 過去世代の成功・失敗から6Gの条件を抽出 |
| `/digest-paper` | 論文を読み3GPP実装制約との差分を記録 |
| `/connect-dots` | トピック間の相乗効果・矛盾・組合せ価値を発見 |
| `/demand-reverse` | ペルソナの課題から技術候補を逆引き |

### コマンド（運用）

| コマンド | 用途 |
|:---|:---|
| `/commit` | 変更をカテゴリ別に分類し適切な粒度でコミット |
| `/review` | 研究ノートを不変原則・6軸の観点でレビュー |
| `/status` | 全ノートの status/confidence 一覧 |
| `/weekly-summary` | 週次研究サマリーの生成 |
