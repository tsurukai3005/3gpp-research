# NR Frame Structure — Bandwidth Part & Carrier Aggregation (TS 38.211 Section 4)

> **対象仕様**: 3GPP TS 38.211 — NR; Physical channels and modulation
> **対象セクション**: Section 4.4.5 "Bandwidth part", Section 4.5 "Carrier aggregation"
> **スコープ**: 物理層リソース構造における BWP と CA の定義・相互関係

---

## 1. 全体の位置づけと TS 38.101-1 との関係

### TS 38.211 Section 4 の役割

TS 38.211 Section 4 "Frame structure and physical resources" は、NR 物理層の**時間・周波数リソースの基本構造**を定義する。TS 38.101-1 Section 5 が「どの周波数帯でどれだけの幅を使えるか」（規制・RF 要件視点）を規定するのに対し、TS 38.211 Section 4 は「その周波数リソースを物理層としてどう構造化するか」（信号処理視点）を規定する。

### 2つの仕様の分担関係

```
┌──────────────────────────────────────────────────────────────────┐
│  TS 38.101-1 Section 5 — 「何が使えるか」（RF / 規制要件）        │
│                                                                  │
│  ・Operating Band: 規制上使える周波数範囲 (n1, n78, ...)         │
│  ・Channel Bandwidth: キャリアが占有する帯域幅 (5〜100 MHz)      │
│  ・ガードバンド / ラスター: 周波数配置の物理的制約               │
│  ・バンドコンビネーション: CA/DC で許される組合せ                │
└────────────────────────────────┬─────────────────────────────────┘
                                 │ 「使える帯域」が決まった上で
                                 ▼
┌──────────────────────────────────────────────────────────────────┐
│  TS 38.211 Section 4 — 「どう構造化するか」（物理層設計）         │
│                                                                  │
│  ・Resource Grid: 周波数軸のリソースブロック (CRB/PRB)           │
│  ・BWP: UE が実際にアクセスする RG のサブセット                  │
│  ・Carrier Aggregation: 複数キャリアの物理層での扱い             │
│  ・Numerology (μ): SCS と時間構造の決定                          │
└──────────────────────────────────────────────────────────────────┘
```

---

## 2. 前提知識: リソースグリッドの階層構造

BWP と CA を理解するために、TS 38.211 Section 4.4 で定義されるリソースグリッドの階層を整理する。

### 周波数軸の階層

```
Operating Band (TS 38.101-1)
 例: n78 = 3300〜3800 MHz (500 MHz 幅)
    │
    ▼
Channel Bandwidth (TS 38.101-1)
 例: 100 MHz (オペレーターの免許帯域)
    │
    ▼
Transmission BW Configuration (TS 38.101-1 Table 5.3.2)
 例: N_RB = 273 RB (100 MHz @ 30 kHz SCS, ガードバンド除外後)
    │
    ▼
Resource Grid — CRB (Common Resource Block)
 Point A を基準に番号付け。ニューメロロジー μ ごとに独立
    │
    ▼
Bandwidth Part (TS 38.211 Section 4.4.5)
 CRB の連続サブセット。UE が実際に動作する範囲
    │
    ▼
PRB (Physical Resource Block)
 BWP 内のローカル番号付け (0 〜 N_size,BWP - 1)
```

### Point A: 全ての周波数参照の起点

- Point A は CRB 0 のサブキャリア 0 の中心周波数
- PCell DL では `offsetToPointA` パラメータで指定（15 kHz SCS 単位の RB オフセット）
- CRB 番号は Point A から上方に連続して振られる
- 各ニューメロロジーで独立した CRB 番号系列が存在する

## 3. Section 4.4.5 Bandwidth Part（BWP）

### 3.1 定義と背景（Why）

#### 正式定義

> "A contiguous set of physical resource blocks, selected from a contiguous subset of the common resource blocks for a given numerology μ on a given carrier."
> — TS 38.211 Section 4.4.5

#### なぜ BWP が必要とされたか

NR は LTE と根本的に異なる設計思想を持つ。**LTE ではセル帯域幅 = UE 帯域幅**（1:1 対応）だったが、NR ではセル帯域幅が最大 100 MHz（FR1）/400 MHz（FR2）に拡大し、全 UE が常に全帯域をモニタすることは非現実的になった。

BWP はこの問題を解決するために Rel-15 で導入された**UE の動作帯域幅を動的に制御する仕組み**である。

| 課題 | BWP による解決 |
|:-----|:-------------|
| 100 MHz 全体をモニタする消費電力 | 狭い BWP に切替えて RF サンプリングレートを低減 |
| 低コスト UE（IoT 等）が広帯域セルに参加できない | 20 MHz BWP のみサポートで参加可能 |
| 異なる SCS のサービス混在 | BWP ごとに異なるニューメロロジーを割当 |
| 周波数ドメインの動的リソース管理 | BWP 切替えで UE の動作帯域を時間変動 |

#### LTE との対比

```
LTE:   Cell BW = UE BW (固定)
       ┌────────────────────────────┐
       │  UE は常に全帯域をモニタ    │
       └────────────────────────────┘

NR:    Cell BW >> UE Active BWP (可変)
       ┌────────────────────────────────────────────────────┐
       │            Cell Bandwidth (100 MHz)                 │
       │    ┌──────────┐                                    │
       │    │Active BWP│  ← UE はここだけモニタ             │
       │    │ (20 MHz) │                                    │
       │    └──────────┘                                    │
       └────────────────────────────────────────────────────┘
```

### 3.2 技術要点（What）

#### BWP の定義パラメータ

各 BWP は以下の2つのパラメータで完全に特定される:

| パラメータ | 意味 |
|:-----------|:-----|
| **N_start,BWP,i** | BWP i の開始 CRB インデックス |
| **N_size,BWP,i** | BWP i の連続 RB 数 |

制約条件:
```
N_start,grid ≤ N_start,BWP,i < N_start,grid + N_size,grid
N_start,grid < N_start,BWP,i + N_size,BWP,i ≤ N_start,grid + N_size,grid
```

#### CRB → PRB 変換

BWP 内の PRB 番号とキャリア全体の CRB 番号の変換:
```
n_CRB = n_PRB + N_start,BWP,i
```

PRB は BWP 内でローカルに 0 から番号付けされ、CRB はキャリア全体で Point A から番号付けされる。

#### BWP の種別

| 種別 | 説明 | 設定方法 |
|:-----|:-----|:---------|
| **Initial BWP (#0)** | 初期アクセス用。CORESET#0 から導出。24〜96 PRB | MIB/SIB1 |
| **First Active BWP** | RRC 設定後に最初にデータ転送する BWP | RRC Reconfiguration |
| **Default BWP** | 非活動タイマー満了時のフォールバック先 | RRC (省略時は Initial BWP) |
| **Dedicated BWP** | サービス固有の BWP | RRC Reconfiguration |

#### 設定上限

- **UL/DL 各方向で最大 4 つの BWP** を設定可能（1キャリアあたり）
- ただし**同時にアクティブにできるのは1つだけ**（各方向・各キャリア）

### 3.3 BWP スイッチング

#### スイッチングの方式

| 方式 | 遅延 | トリガー | 用途 |
|:-----|:-----|:---------|:-----|
| **DCI ベース** | ~2 ms | DCI Format 0_1/1_1 の BWP Indicator フィールド | 高速切替（スケジューラ判断） |
| **タイマーベース** | タイマー依存 | `bwp-InactivityTimer` 満了 | 省電力（自動フォールバック） |
| **RRC ベース** | ~10 ms | RRC Reconfiguration | 長期的な帯域変更 |
| **MAC 起動** | 可変 | Random Access 手順 | 初期アクセスへの復帰 |

#### スイッチング遅延（TS 38.133）

UE は BWP スイッチング遅延をケイパビリティとして報告:
- **Type 1**: 低遅延（高性能 UE）
- **Type 2**: 高遅延（低コスト UE）

スイッチング中、UE は送受信を停止する（ブラインド期間が発生）。

### 3.4 実装制約（How）

#### RF フロントエンドへの影響

BWP のサイズが変わると、ADC/DAC のサンプリングレートと RF フィルタ帯域が変化する:

```
狭い BWP (20 MHz)           広い BWP (100 MHz)
┌──────────┐               ┌──────────────────────────────────┐
│ ADC: 低速 │               │ ADC: 高速                        │
│ 消費電力: 低│              │ 消費電力: 高                     │
│ 処理負荷: 低│              │ 処理負荷: 高                     │
└──────────┘               └──────────────────────────────────┘
```

#### 制約事項

- DL BWP は SSB 帯域幅以上でなければならない
- 各 DL BWP に最低1つの CORESET（UE-Specific Search Space 付き）が必要
- プライマリ DL BWP には Common Search Space 付き CORESET が必要
- UE はアクティブ BWP 外で PDSCH/PDCCH/CSI-RS/TRS を受信してはならない

## 4. Section 4.5 Carrier Aggregation（物理層観点）

### 4.1 定義と背景（Why）

CA は複数のコンポーネントキャリア（CC）を同時に送受信し、合計スループットを向上させる技術。TS 38.211 Section 4.5 は、CA の**物理層での扱い**を定義する（バンドコンビネーションの許容組合せは TS 38.101-1 で規定）。

#### 物理層での CA の意味

各 CC は完全に独立したリソースグリッドを持つ:
- 独自のニューメロロジー（SCS が異なりうる）
- 独自の BWP 設定（最大4つ/方向/CC）
- 独自のフレームタイミング

### 4.2 技術要点（What）

#### 物理層パラメータ

| パラメータ | 値 | 説明 |
|:-----------|:---|:-----|
| 最大 CC 数 | DL 16 + UL 16 | Rel-15 以降 |
| 合計帯域幅上限 | ~6.4 GHz（理論値） | 16 × 400 MHz (FR2) |
| `ca-SlotOffset` | Higher-layer パラメータ | SCell のスロットタイミングオフセット |

#### ca-SlotOffset の意味

CC 間で SCS が異なる場合、スロット境界が整列しない。`ca-SlotOffset` は PCell に対する SCell のスロットレベルタイミングオフセットを指定し、HARQ タイミングとスケジューリング調整を可能にする。

```
PCell (30 kHz SCS):  |---Slot 0---|---Slot 1---|---Slot 2---|---Slot 3---|
                     0          0.5          1.0          1.5          2.0 ms

SCell (15 kHz SCS):  |------Slot 0------|------Slot 1------|
                     0                  1.0                2.0 ms
                     ↑
                     ca-SlotOffset でオフセット調整
```

### 4.3 「キャリア + BWP」2階層アーキテクチャ

NR の CA は**2段階の活性化制御**を持つ:

```
レベル1: キャリア活性化
┌─────────────────────────────────────────────────────┐
│ PCell (常時アクティブ)                               │
│ SCell 1 (MAC CE でアクティベート/デアクティベート)    │
│ SCell 2 (MAC CE でアクティベート/デアクティベート)    │
└─────────────────────────────────────────────────────┘
                        │
                        ▼
レベル2: BWP 活性化（アクティブなキャリア内でのみ動作）
┌─────────────────────────────────────────────────────┐
│ PCell:  Active DL BWP = BWP#2 (100 MHz, 30 kHz)    │
│ SCell1: Active DL BWP = BWP#1 (50 MHz, 30 kHz)     │
│ SCell2: Active DL BWP = BWP#0 (20 MHz, 15 kHz)     │
└─────────────────────────────────────────────────────┘
```

**重要な規則**: BWP はアクティブなキャリア内でのみ活性化できる。各アクティブ CC は DL/UL それぞれ正確に1つのアクティブ BWP を持つ。

### 全体のアーキテクチャ関係

```

│  ┌─────────────────────────────────────────────────────────────────┐   │

│  │  │  │  │   BWP #1 (Active)    │  │   BWP #2         │   │  │   │   │
│  │  │  │  │   50 MHz, 30kHz SCS  │  │   100MHz, 15kHz  │   │  │   │   │

│  │  │  │  └──────────────────────┘  └──────────────────┘   │  │   │   │

```

### CA + BWP の組合せ動作

```
UE が CA_n1A-n78C をサポートするケース:

  Operating Band n1 (FDD)          Operating Band n78 (TDD)
  ┌───────────────────┐            ┌────────────────────────────────────────┐
  │ CC1: 20 MHz       │            │ CC2: 100 MHz          CC3: 100 MHz    │
  │ ┌───────────────┐ │            │ ┌────────────────┐   ┌──────────────┐ │
  │ │Active BWP     │ │            │ │Active BWP      │   │Active BWP    │ │
  │ │20 MHz (全幅)  │ │            │ │80 MHz          │   │50 MHz        │ │
  │ │SCS: 15 kHz    │ │            │ │SCS: 30 kHz     │   │SCS: 30 kHz   │ │
  │ └───────────────┘ │            │ └────────────────┘   └──────────────┘ │
  └───────────────────┘            └────────────────────────────────────────┘
  PCell                             SCell 1                 SCell 2

  合計アクティブ帯域: 20 + 80 + 50 = 150 MHz
  合計設定帯域:       20 + 100 + 100 = 220 MHz
```

### Operating Bands との接続点

| Operating Bands 側の定義 | BWP/CA 側での利用 |
|:------------------------|:-----------------|
| バンドの周波数範囲 | CC の配置可能位置を決定 |
| Channel Bandwidth テーブル | Resource Grid のサイズ上限を決定 |
| SCS × BW の許容組合せ | BWP のニューメロロジー選択肢を制約 |
| バンドコンビネーション (5.2A) | CA で組み合わせ可能な CC を制約 |
| ガードバンド (5.3.3) | 実効 RB 数（BWP の最大サイズ）を決定 |
| チャネルラスター (5.4.2) | Point A の配置候補を制約 |

## 6. トレードオフの整理

### BWP サイズのトレードオフ

| 観点 | 狭い BWP (例: 20 MHz) | 広い BWP (例: 100 MHz) | 根拠 |
|:-----|:---------------------|:----------------------|:-----|
| 消費電力 | 低い | 高い | ADC サンプリングレートが帯域幅に比例 |
| ピークスループット | 制限される | 最大化 | スケジューリング可能な RB 数に比例 |
| スケジューリング柔軟性 | 制約あり | フルアクセス | 周波数選択性ダイバーシティが広い方が有利 |
| UE RF 複雑さ | 低い | 高い | 広帯域フィルタ・ADC が必要 |
| BWP 切替頻度 | 高い（広い BWP が必要な場面で切替発生） | 低い（常に広い BWP で運用） |


### BWP スイッチングのトレードオフ

| 方式 | メリット | デメリット |
|:-----|:---------|:----------|
| DCI ベース (~2ms) | 高速応答、スケジューラが最適判断 | PDCCH リソース消費、頻繁な切替はオーバヘッド |
| タイマーベース | 自動的、シグナリング不要 | 応答遅れ、タイマー値の最適設定が困難 |
| RRC ベース (~10ms) | 長期的な構成変更に適切 | 遅延大、頻繁な切替には不向き |

## 7. 前世代との差分

### LTE → NR の進化

| 側面 | LTE | NR | 進化の理由 |
|:-----|:----|:---|:-----------|
| セル BW と UE BW の関係 | 1:1 固定 | 独立（BWP で分離） | 広帯域セル + 多様な UE |
| SCS | 15 kHz 固定 | 15/30/60/120/240 kHz | 多様なサービス要件 |
| キャリア内の SCS 混在 | 不可 | BWP ごとに異なる μ 可能 | eMBB + URLLC 共存 |
| CA の最大 CC 数 | 5 (Rel-10) → 32 (Rel-14) | 16 (Rel-15) | 実装複雑さとのバランス |
| 動的帯域幅適応 | なし（RRC による再設定のみ） | BWP switching (DCI/Timer) | UE 省電力の重要性増大 |
| CA + 帯域幅適応 の組合せ | 限定的 | CC ごとに独立 BWP 管理 | きめ細かなリソース制御 |

### なぜ LTE の固定帯域幅モデルが限界だったか

LTE では 20 MHz が最大チャネル帯域幅であり、全 UE が常に全帯域をモニタしても消費電力は許容範囲だった。NR で 100 MHz/400 MHz に拡大した結果:

1. **消費電力問題**: 広帯域 ADC の常時動作は電池寿命を著しく短縮
2. **UE カテゴリの多様化**: IoT デバイスから XR ヘッドセットまで、帯域幅要件が大幅に異なる
3. **周波数効率**: UE が使わない帯域を常にモニタするのは無駄

BWP はこれらを**物理層レベルで**解決する NR 固有の仕組みである。

## 8. 未解決課題と知財余地

### BWP 関連の課題

| 課題 | 状況 | 知財余地 |
|:-----|:-----|:---------|
| BWP 切替遅延の最適化 | Type 1/2 の2段階のみ → より細かな制御が可能か | UE 実装の工夫で遅延短縮 |
| bwp-InactivityTimer の最適値選定 | トラフィックパターン依存、適応的制御は FFS | AI/ML ベースの予測切替 |
| CA + BWP の同時切替制御 | 複数 CC での協調 BWP 切替の最適化 | クロスキャリア BWP 管理 |
| RedCap UE の BWP 制約 | Rel-17 で 20 MHz BWP に制限 → さらなる低減は？ | 5 MHz BWP の実現可能性 |
| BWP と省電力の定量評価 | 切替頻度 vs 消費電力削減のトレードオフモデル | 実測ベースの最適化アルゴリズム |

### CA 物理層の課題

| 課題 | 状況 | 知財余地 |
|:-----|:-----|:---------|
| 異 SCS CC 間のタイミング管理 | ca-SlotOffset で対応 → 高精度化の余地 | サブスロットレベル同期 |
| 16CC 運用時の制御チャネル設計 | PDCCH 容量圧迫 → クロスキャリアスケジューリング最適化 | CC グルーピング戦略 |
| CA + BWP 組合せ爆発 | テスト負荷増大 → UE ケイパビリティ報告の簡素化 | ケイパビリティ圧縮手法 |
