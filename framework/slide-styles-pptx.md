# PPTX スライドスタイル運用ガイド

`.pptx` を生成・改変するときのスタイル規約。実装は [`tools/pptx/`](../tools/pptx/) を参照。TeX/Beamer 系の規約は本ファイルでは扱わない（[`framework/templates/slide-tex/README.md`](./templates/slide-tex/README.md) を参照）。

## 1. スタイルは1個に統一する

スタイルは `default` のみ。定義ファイルは [`tools/pptx/styles-meta/default.yaml`](../tools/pptx/styles-meta/default.yaml)。

色・レイアウト・フォントを一元管理し、スライド作成時にコード側で見出しバー・フッタ・参考文献欄・callout を描画する。

## 2. 色

| 役割 | 色 | 使用箇所 |
|:---|:---|:---|
| Main | `#CC0033` (R204 G0 B51) | 表紙・目次の全面塗り、見出しバー塗り、本文の `**強調**`、表ヘッダ塗り、箇条書きの先頭マーカー |
| Assort | `#5B6B7E`（青みがかった灰） | フッタ文字（資料タイトル・ページ番号） |
| Accent | `#E8ECF1`（Assort の淡色） | callout 塗り、表偶数行のゼブラ |
| 文字色 | `#1F1F1F` | 本文 |
| 背景 | `#FFFFFF` | 本文ページ全面 |

callout は **すべて Accent (#E8ECF1) + 文字 #1F1F1F** に統一する。tone による色分けは廃止（後方互換のため `tone` フィールドを spec に書いても許容するが、描画は accent のみ）。

## 3. フォント

- 全テキストで **メイリオ (Meiryo)** を使用。サイズ既定値:
  - 見出しバー: 22pt 太字
  - 本文・callout: 18pt（行間 1.25 / callout は 1.15）
  - 表紙タイトル: 40pt 太字 / サブタイトル: 20pt / 日付: 14pt
  - 目次タイトル: 32pt 太字 / 目次エントリ: 20pt
  - フッタ・参考文献: 10pt

## 4. レイアウト

```
本文ページ:
┌──────────────────────────────────────────────────────────┐
│ [Main 塗り] タイトル                       [参考文献欄] │  ← 見出しバー (h=0.7")
├──────────────────────────────────────────────────────────┤
│                                                          │
│  本文（箇条書き → callout の順、または callout のみ）    │
│                                                          │
├──────────────────────────────────────────────────────────┤
│ 資料タイトル                                  n / N      │  ← フッタ (h=0.35")
└──────────────────────────────────────────────────────────┘
       16:9 / 13.333 × 7.5 inch

表紙・目次:
┌──────────────────────────────────────────────────────────┐
│                                                          │
│                  [Main 全面塗り]                         │
│             白文字・中央配置のタイトル                   │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

- 見出しバー: 上端 0..0.7"、Main 色塗り、白タイトル（22pt 太字）
- 参考文献欄: バー右端 3.0" 幅、白文字 10pt 右寄せ。`reference` を指定したスライドにだけ表示
- 本文領域: バーの下 0.2" から始まり、フッタの 0.1" 上で終わる
- フッタ: 左に `metadata.title`（10pt Assort）、右に `n / N`（10pt Assort）。**表紙・目次はフッタを持たない**

## 5. 本文の構成要素

### 5.1 箇条書き

文字列を `body` に並べると、先頭に Main 色の `▪` マーカーが付く 18pt の箇条書きになる。行間は 1.25。

```json
"body": [
  "サブキャリア間隔 Δf は **15·2^μ kHz** で離散化される",
  "スロット長は **2^μ** に反比例して短くなる"
]
```

- `**…**` で囲んだ部分は Main 色（赤）の太字になる。文字強調はこの形式のみ。
- 連続する箇条書きは1つのテキストフレームにまとまる。
- **箇条書きブロックの高さは行数ベースで自動計算される**。残り領域を全部使い切らない（callout が直下に詰めて配置される）。

### 5.2 callout（塗りつぶしテキストボックス）

結論・前提・補足など、強調表示する1行を入れる。

```json
{"kind": "callout", "text": "結論: 一行で言い切る"}
```

- 固定高さ 0.55"。**1行で言い切る** こと。長文は箇条書き側で書く。
- 連続して並べると stacked になる。
- 色は Accent（薄い灰色）固定。`tone` を書いても無視される。

## 6. 参考文献欄の使い方

スライド内容の根拠（仕様セクション・論文番号・Tdoc）を `reference` に書くと、見出しバー右端に小さく表示される。

```json
{"type": "content", "title": "...", "reference": "TS 38.211 §4.3.2", "body": [...]}
```

- 1〜2 個の出典を簡潔に。複数なら `;` で区切る
- 根拠の本文は別途ノートまたは箇条書きで詳述する
- 全スライドに付ける必要はないが、技術主張のあるスライドには付けるのが望ましい

## 7. 表紙・目次・セクション扉

### 7.1 表紙 (`type: "title"`)

- 全面 Main 色塗り、白文字
- タイトル / サブタイトル / 日付 を縦中央スタックで中央寄せ
- **著者名 (`author`) は表示しない**（spec に書かれていても描画されない）
- フッタ・ページ番号なし

### 7.2 目次 (`type: "toc"`)

- 全面 Main 色塗り、白文字
- 上部にタイトル（既定「目次」）
- 中央付近に `entries` を箇条書き表示（`「1. ...」` 形式）
- フッタ・ページ番号なし
- **自動生成しない**: spec で明示する。section スライドの全タイトルを並べると多すぎる場合があるため、発表ストーリー側で取捨選択する

```json
{
  "type": "toc",
  "title": "目次",
  "entries": [
    "Operating Bands と帯域配置",
    "Subcarrier Spacing と OFDM 内部構造",
    "BWP と CA — UE への割当戦略"
  ]
}
```

### 7.3 セクション扉 (`type: "section"`)

中央に厚い赤バー、白タイトル。本文なし。フッタあり。

## 8. 表

- ヘッダ行: Main 色塗り、白文字太字、上下中央
- 本文行: 奇数行=白、偶数行=Accent 色（`#E8ECF1`）でゼブラ。**セルテキストは上下中央配置**
- 行高は内容ベースで計算（既定 0.45"）。本文領域に収まらない場合のみ均等縮小
- 列幅は均等（必要なら生成後に PowerPoint で調整）

## 9. 出力 PPTX の git 管理

- 出力先: `slides/pptx/<title>.pptx`
- バイナリだが **git 追跡対象**（再現性より履歴の追跡を優先）
- 巨大化が顕在化したら個別に `.gitignore` で外す方針を再検討

## 10. 修正・追記のフロー

| やりたいこと | コマンド | 備考 |
|:---|:---|:---|
| 新規ビルド | `build_slides.py build <spec.json>` | エージェントが文脈から spec を組む |
| 既存 pptx の状態を確認 | `build_slides.py inspect <pptx>` | JSON で構造取得 |
| 末尾に追加 | `apply <pptx>` + `{"op":"append", "slide":{...}}` | 単発でも複数でも |
| 指定位置に挿入 | `{"op":"insert","index":N,"slide":{...}}` | 1-origin |
| スライド差替え | `{"op":"replace","index":N,"slide":{...}}` | 元スライドは削除 |
| 削除 | `{"op":"delete","index":N}` |  |
| 並べ替え | `{"op":"move","from_index":A,"to_index":B}` |  |
| 文字列置換 | `{"op":"update_text","index":N,"find":"旧","replace":"新"}` | 本文・表・ノート横断 |
| スタイル一覧 | `build_slides.py list-styles` | YAML から抽出 |

PowerPoint で直接手を入れた pptx も `inspect` で読み戻せる。エージェントが現状把握→操作生成→`apply` で再書き込みする。
