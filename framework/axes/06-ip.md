# 軸6: 知財軸

**問い: この技術は特許化でき、標準必須特許（SEP）として権利化できるか？**

> 親: [[README|axes（軸 MoC）]] | 兄弟軸: [[01-technology-layer|軸1]] / [[02-generation|軸2]] / [[03-value|軸3]] / [[04-market|軸4]] / [[05-adoption-factors|軸5]]
> 関連テンプレ: [[../templates/gap-analysis|ギャップ分析]]（実装制約差分の言語化）


## 3つの評価観点

### 新規性
- 既存論文・Tdoc・公開特許公報で類似アイデアはあるか？
- 3GPP ポータルで関連 Tdoc を検索し、既出のアプローチを確認する
- arXiv / IEEE Xplore で先行研究を確認する

### 進歩性（実装制約差分）

**ここが最も重要。論文の理想仮定と 3GPP の実装制約の差が進歩性の源。**

| 論文の理想仮定 | 3GPP の実装制約 |
|:---|:---|
| 理想チャネル（AWGN, i.i.d. Rayleigh） | 実際の同期ズレ、遅延拡散、ドップラー |
| 連続値パラメータ | DCI の数ビット量子化 |
| 完全 CSI（チャネル完全既知） | 限られたフィードバック帯域 |
| 無限計算資源 | チップ面積・消費電力制約 |
| 単一セルモデル | マルチセル干渉環境 |
| 完全同期 | 非理想的バックホール、タイミングオフセット |

### 規格マッピング可能性
- TS の必須要件（"shall" 文言）に落とし込めるか？
- クレームの構成要件が仕様書の記述と対応するか？
- 実装のバリエーション（代替パラメータ、フォールバック手順）を明細書に記載できるか？

## 問いかけ

- 既存の Tdoc や論文で類似アイデアはあるか？
- 論文の理想仮定と 3GPP の実装制約の差はどこにあるか？
- 規格の "shall" 文言に対応する形でクレームを書けるか？
- 明細書に記載すべきバリエーション（代替案、パラメータ範囲）は何か？
- 競合他社が提案しそうな代替アプローチは何か？
