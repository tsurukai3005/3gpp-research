# テンプレート: ギャップ分析

スキル `/analyze-gap` から参照される。

## 3つのバケット

[トピック名] について、以下の3バケットでギャップを整理する。

### Gap A: 学術で解決済みだが 3GPP に未取込

- 学術論文で提案・検証されているが、3GPP の SI/WI にまだ反映されていない技術
- 「なぜ取り込まれていないか」の理由を推定する
  - 実装複雑性？ コンセンサス未形成？ タイミング？

### Gap B: 3GPP FFS に残っており学術でも議論中

- Chairman's Notes の FFS 項目として残っている課題
- 学術界でも活発に研究されている
- **特許創出の最短距離**（戦略文書5.3節）

### Gap C: 実装制約ギャップ

- 論文の理想仮定と 3GPP の実装制約の間にある差
- 同期ズレ、後方互換性、シグナリング負担、チップ制約
- **進歩性の最大の源泉**（原則2）

## 各ギャップへの問い

各ギャップについて以下を記載する:

1. **ギャップの内容**: 何が解決されていないか
2. **出典**: 論文、Tdoc、Chairman's Notes の具体的な参照
3. **該当する軸**: 6軸のどれに関連するか
4. **知財評価**: 進歩性の源泉として強いか弱いか
5. **Next Steps**: このギャップを深掘りするために次に調べるべきこと
