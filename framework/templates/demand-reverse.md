# テンプレート: 需要からの逆引き

スキル `/demand-reverse` から参照される。

## 入力

- ペルソナファイル: `framework/personas/` のファイル
- ペインポイント: 自由記述（例: 「カバレッジコスト」「エネルギー消費」）

## 分析の手順

### 1. ペルソナのコンテキスト把握

指定ペルソナファイルから以下を抽出:
- ペインポイントの一覧
- 重視する価値軸
- 技術への関心領域

### 2. トピック候補のスキャン

`documents/` の全ノート一覧をスキャンし、各ノートの:
- 価値軸（axes.value）
- 市場軸（axes.market）
がペルソナのペインポイントに合致するかを評価する。

### 3. 適合度ランキング

候補技術を以下の基準で評価し、ランク付けする:

| 基準 | 問い |
|:---|:---|
| 直接的関連性 | そのペインポイントを直接解決するか？ |
| 成熟度 | 3GPP での議論段階（SI/WI/TS化済み） |
| 実装実現性 | 当該ペルソナの制約（コスト、インフラ）で実現可能か？ |
| 代替手段の有無 | 既存の代替技術と比較して優位か？ |

### 4. 調査提案の生成

適合度の高い技術について、次に調べるべき情報源を具体的に提案する。

```markdown
## 調査提案
- [ ] [技術A] について arXiv で "[キーワード]" を検索
- [ ] [技術B] の 3GPP での SI/WI 状況を確認（Tdoc: R1-XXXXXXX）
- [ ] NGMN の [文書名] で [ペルソナ地域] の需要データを確認
```

### 5. 未カバー領域の特定

`documents/` にまだノートがないが、このペルソナのペインポイントに
関連しそうな技術領域を特定し、新規調査の候補として提示する。

## 出力

チャットに出力。保存する場合は `documents/YYYY-MM-DD_需要逆引き-persona-slug.md`
